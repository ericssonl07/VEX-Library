\doxysubsection{Detailed Description}
The {\ttfamily \doxylink{class_odometry}{Odometry}} class tracks the position and rotation of the robot. 

\begin{DoxyAttention}{Attention}
The {\ttfamily \doxylink{class_odometry}{Odometry}} class will only function within scope, so make sure it is managed by another object. By default, {\ttfamily \doxylink{class_chassis}{Chassis}} instances (and all instances of derived classes) manages the {\ttfamily \doxylink{class_odometry}{Odometry}} object. 
\end{DoxyAttention}

int Odometry\+::thread\+\_\+sleep\hspace{0.3cm}{\ttfamily [private]}}



The time in milliseconds to sleep between iterations. This is used to prevent resource exhaustion. 

If the thread sleep is too low, the CPU will be overworked with all resources dedicated to the tracking function, which will cause the program to freeze; other threads will not be able to run. If the thread sleep value is too high, the tracking function will experience a high degree of granularity, which may cause tracking inaccuracies. By default, it is set to 10 milliseconds, which is a good balance between high granularity and resource efficiency. \Hypertarget{class_odometry_a03cab013e84717fd10e1f4af40aa5fcb}\index{Odometry@{Odometry}!tracking@{tracking}}
